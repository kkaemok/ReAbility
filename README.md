# ReAbility

Paper 1.21.x 서버용 능력 약탈/길드/PvP 플러그인입니다.  
플레이어마다 능력을 부여하고, 능력별 패시브/스킬/경제/텔레포트/길드 시스템을 통합 관리합니다.

## 1) 프로젝트 개요

- 플러그인 메인: `org.kkaemok.reAbility.ReAbility`
- 핵심 데이터: `playerdata.yml`, `guilds.yml`, `user-shop.yml`, `abilities/*.yml`
- `playerdata.yml`: 능력, 만료시간, 코인, 채굴 카운트, 영혼/투자 상태
- `guilds.yml`: 길드 정보/정원
- `user-shop.yml`: 유저 상점 등록 목록
- `abilities/*.yml`: 능력별 스킬 코스트/쿨타임 오버라이드
- 주요 시스템: 능력 랜덤 배정, 리롤권, 전투 태그, 텔레포트, 길드, 코인 경제, 상점, 점수판

## 2) 요구사항

- Java 21
- Paper `1.21.11` 계열
- 필수 플러그인: LuckPerms, ProtocolLib
- 선택 플러그인: `nicknames` (있으면 닉네임 API 연동)

## 3) 빌드/실행

```bash
./gradlew build
```

개발 서버 실행(Gradle run-paper):

```bash
./gradlew runServer
```

## 5) 능력 시스템 상세

### 5-1. 등급

`D`, `C`, `B`, `A`, `S`, `S+`, `SS`

- Enum 기본 지속시간: `D/C/B=1일`, `A=2일`, `S=3일`, `S+=4일`, `SS=5일`
- 기본 `config.yml`은 모든 등급을 `24시간`으로 오버라이드
- `S/S+/SS`는 공통 버프 `저항1 + 재생1` 자동 적용

### 5-2. 자동 능력 배정

접속 시 능력이 없거나 만료 상태면 아래 가중치로 랜덤 배정합니다.

| 등급 | 확률 |
|---|---|
| D | 45.0% |
| C | 40.0% |
| B | 9.0% |
| A | 5.0% |
| S | 0.86% |
| S+ | 0.04% |
| SS | 0.10% |

추가로 주기 태스크(`AbilityUpdateTask`)가 만료 능력을 감지하면 즉시 `D등급` 재배정도 수행합니다.

### 5-3. 리롤권 확률

| 리롤권 | 확률 |
|---|---|
| C등급 이하 리롤권 | D 60 / C 40 |
| B등급 이상 I (`B1`) | B 60 / A 35 / S 4.7 / S+ 0.1 / SS 0.2 |
| B등급 이상 II (`B2`) | B 45 / A 48 / S 6.35 / S+ 0.15 / SS 0.5 |
| A등급 이상 | A 81 / S 17.5 / S+ 0.5 / SS 1.0 |
| S등급 이상 | S 93 / S+ 1 / SS 6 |

### 5-4. 스킬 트리거 규칙

- 기본 트리거는 웅크리기(`PlayerToggleSneakEvent`)입니다.
- 일부 능력은 우클릭/타격/사망/월드시간/거리 기반 이벤트로 동작합니다.
- `sneak-skill.block-look-guard.enabled=true`면 상호작용 가능한 블록을 바라볼 때 스킬 발동을 차단합니다.

### 6-1. 최근 밸런스 반영 (2026-04-02)

- 영역술사: 기본 반경 200, 영역 확장 시 반경 500(30분), 확장 중 저항1 추가
- 셰프: `대식가` 지속 1분/쿨 1분, 같은 길드에 `EATER`가 있으면 `대식가` 쿨타임 제거
- 셰프: 양고기/마황 체력 증가를 `+2줄` 기준으로 조정, 돼지고기 폭발 면역 2분
- 발키리: 도끼 공격속도 `-45%`, 회오리 중심 피해 `초당 25`
- 조커: `nicknames` 감지 시 이름표/탭리스트/채팅을 API 기준으로 가명 동기화, 실패 시 ProtocolLib 폴백
- 조커: `스파이`는 길드채팅뿐 아니라 귓속말(`/msg`, `/tell`, `/w`, `/whisper`, `/m`)도 감지
- 공간 지배자: RTP 쿨타임 0초, RTP 범위 ±30000, RTP 직후 30초 관전자(유령) 상태
- 미녀: 같은 길드원이 미녀를 볼 때는 정지 효과 미적용, 다른 길드/무길드 상대만 적용
- 위 수치는 기본값이며 `plugins/ReAbility/abilities/*.yml`에 기존 값이 있으면 해당 값이 우선합니다.

### 6-2. 최근 기능 반영 (2026-04-08)

- D 등급 신규 추가: `HUSK`, `DISRUPTOR`, `MICHAEL_JACK`, `FASHION_LEADER`
- C 등급 신규 추가: `TREASURE_APPRAISER`, `GANGGANGYAKYAK`, `BULLDOZER`
- 방해꾼(`DISRUPTOR`) 스킬 방해 판정을 웅크리기/우클릭 기반 스킬 트리거에 연동
- 마이클잭(`MICHAEL_JACK`)은 크로노스 경직(`시간 단절`) 대상에서 제외

## 7) 네크로맨서 영혼 스킬 트리

네크로맨서는 `/영혼 상점`에서 영혼을 투자해 스킬을 해금합니다.  
영혼은 몹 처치 시 획득하고 기본 `1시간` 후 만료됩니다.

| 스킬 | 키 | 해금 영혼 | 아이콘 |
|---|---|---:|---|
| 영혼 수확 | `soul_harvest` | 50 | `SOUL_LANTERN` |
| 소울 나이트 | `death_knight` | 100 | `NETHERITE_INGOT` |
| 죽음의 본능 | `death_instinct` | 300 | `DIAMOND_BLOCK` |
| 스컬크 소울 | `sculk_soul` | 1000 | `SCULK_CATALYST` |
| 죽음의 꽃 | `death_flower` | 2000 | `WITHER_ROSE` |
| 세계의 종말 | `end_of_world` | 10000 | `CRYING_OBSIDIAN` |

추가 동작(코드 기준):

- 소울 나이트: 소비 영혼량에 따라 버프 강도/지속시간 증가
- 죽음의 본능: 최근 사망 위치 복귀(조건/재료 필요)
- 스컬크 소울: 워든 처치 이후 윈도우 내 강화 변신 루트
- 죽음의 꽃: 장판/흡혈/반사형 교전 효과
- 세계의 종말: 단계형 아포칼립스(시간 경과에 따른 야간 고정/광역 압박)

## 8) 전투/텔레포트/월드 시스템

### 8-1. 전투 태그

- 플레이어 간 피격 시 `15초` 전투 상태
- 전투 중(OP 제외) 모든 명령어 차단
- 전투 중 로그아웃 시 즉사 처리
- 액션바 카운트다운 출력

### 8-2. 텔레포트

- `TPA`/`TPHere` 요청-수락 구조, 요청 만료시간 설정 가능
- 수락 후 3초 카운트다운 이동
- 전투 중 요청/이동 불가
- 기본 쿨타임(설정 가능): TPA 180초(`HOMELOVER` 60초), RTP 600초(`SPACE_RULER` 0초)
- RTP 범위: 기본 ±10000, `SPACE_RULER`는 ±30000

### 8-3. 월드/밸런스 기본값

- 기본 최대 체력: `40.0`
- 도끼 피해 보정: `-2.0`
- 월드보더: `(-30000, -30000) ~ (30000, 30000)`
- 채굴 제한: 다이아 시간당 100개(`MINER` +600), 고대 잔해 3시간당 2개(`MINER` +5)

## 9) 경제/상점/길드

### 9-1. 코인 경제

- 플레이 시간 1시간마다 `+10 코인` 자동 지급
- `/능력 송금`으로 유저 간 전송
- `/레능 코인`, `/레능 코인설정`으로 관리자 조정

### 9-2. 기본 상점 (`/상점`)

- `config.yml` 기반 카테고리/아이템/가격/로어 완전 구성형 GUI
- MiniMessage 표시 지원
- 구매 시 코인 차감 후 아이템 지급(인벤 초과분은 바닥 드랍)

### 9-3. 유저 상점 (`/유저상점`, `/팔기`, `/가격설정`)

- 판매 전용 GUI에서 아이템 임시 등록 후 가격 확정
- 페이지형 마켓 조회 및 구매
- 구매 시 구매자 코인 차감 + 판매자 코인 지급
- 목록은 `user-shop.yml`에 영속 저장

### 9-4. 길드 시스템

- 길드 생성 비용: 다이아 100개
- 가입 요청/수락, 길드 탈퇴, 길드챗 토글
- 길드 정원 확장: 5명 시작, 최대 10명
- LuckPerms 그룹 자동 생성(`guild_<name>`) + 접미사 적용
- 길드챗은 같은 길드원 + OP가 수신

## 10) 닉네임/가명 연동

- `nicknames` 플러그인이 감지되면 API 브리지 활성화
- 조커 가명은 `nicknames` API 경유로 이름표/탭리스트/채팅 표시까지 동기화 시도
- `nicknames` API 경로가 실패하면 ProtocolLib 기반 가명 패킷으로 폴백
- 명령어 입력 시 가명을 실제 닉으로 치환(`JokerCommandAliasListener`)

## 11) 설정 포인트

`config.yml` 핵심 경로:

- `world-settings.*`
- `mining-limits.*`
- `ability-settings.*`
- `teleport-settings.*`
- `sneak-skill.block-look-guard.*`
- `messages.*`
- `shop.*`

능력별 세부 튜닝:

- `plugins/ReAbility/abilities/<ABILITY_KEY>.yml` 자동 생성
- 예: `skills.<skill>.cooldown-ms`, `skills.<skill>.cost.item`, `skills.<skill>.cost.amount`

## 12) 개발 참고

- 이벤트 등록: `ReAbility#registerEvents`
- 명령어 등록: `ReAbility#registerCommands`
- 능력 등록: `AbilityManager#registerAbilities`
- 코드상 사용하지 않는 레거시 커맨드 클래스(`MainCommand`, `KeepCommand`, `BarrierCommand`)가 남아 있으나 현재 `plugin.yml`에 바인딩된 명령은 본 문서 기준입니다.
